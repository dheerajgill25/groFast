## Parent story — E02-Th02-S01

**Title:** As a catalog manager, I want SKU/category CRUD with bulk upload, and as a customer I want honest per-store browse and detail, so that the storefront is never wrong

**User story:** As a catalog manager, I want SKU/category CRUD with bulk upload, and as a customer I want honest per-store browse and detail, so that the storefront is never wrong

**Acceptance criteria:**

- [ ] Only in-stock, priced items for the serving store render in browse

- [ ] Age-restricted/regulated categories cannot publish (BR-9) via UI or CSV
- [ ] PDP shows price/discount/pack/availability matching the DB of record

## Test cases this task must make pass

### TC-FUNC-006 — Category browse shows only in-stock priced items of serving store
- **Type:** integration
- **Requirement:** FR-4
- **Preconditions:** Store A: SKU-1 in stock+priced, SKU-2 OOS, SKU-3 unpriced
- **Steps:** 1. Open category as store-A customer
- **Expected:** Only SKU-1 renders; response from cache path; server time recorded
- **Test data:** `fixture `catalog/store_a.sql``
- **Priority:** P0

### TC-FUNC-008 — PDP shows correct price/discount/pack/availability
- **Type:** integration
- **Requirement:** FR-6
- **Preconditions:** SKU with MRP ≠ selling price at serving store
- **Steps:** 1. Open product detail
- **Expected:** Price, discount %, pack size, images, availability at serving store all match DB of record
- **Priority:** P1

### TC-FUNC-025 — Catalog CRUD live within 60 s; bulk CSV upload
- **Type:** integration
- **Requirement:** FR-21
- **Preconditions:** Catalog-role admin
- **Steps:** 1. Edit SKU price + image 2. Upload 200-row CSV 3. Poll storefront
- **Expected:** Storefront (cache + Meili) reflects changes ≤ 60 s; CSV rows applied with success report
- **Test data:** `fixture `catalog/bulk_200.csv``
- **Priority:** P0

### TC-NEG-004 — OOS/unpriced SKUs never render in category
- **Type:** integration
- **Requirement:** FR-4
- **Preconditions:** SKU flips to OOS mid-session
- **Steps:** 1. Browse category 2. Invalidate stock 3. Refresh after propagation
- **Expected:** OOS SKU absent (or marked, per design) after ≤ 60 s propagation; no ghost inventory purchasable
- **Priority:** P0

### TC-NEG-006 — SKU unavailable at serving store blocks add-to-cart
- **Type:** integration
- **Requirement:** FR-6
- **Preconditions:** SKU stocked at store B only; customer served by store A
- **Steps:** 1. Open PDP via deep link
- **Expected:** Shows unavailable-at-your-store state; add-to-cart disabled
- **Priority:** P1

### TC-NEG-025 — Malformed bulk CSV rejected row-wise
- **Type:** integration
- **Requirement:** FR-21
- **Preconditions:** CSV: 5 valid rows, 3 malformed (bad price, missing SKU, bad category)
- **Steps:** 1. Upload
- **Expected:** Valid rows applied; 3 rejects reported with row number + reason; no partial-row corruption
- **Test data:** `fixture `catalog/bulk_bad.csv``
- **Priority:** P1

### TC-NEG-038 — Age-restricted/regulated item cannot publish to V1 catalog
- **Type:** unit
- **Requirement:** BR-9
- **Preconditions:** SKU with restricted category flag
- **Steps:** 1. Attempt publish via admin + bulk CSV
- **Expected:** Publish blocked in both paths with compliance reason; SKU never reaches storefront/index
- **Priority:** P1

### TC-PERF-001 — Browse p95 < 200 ms at launch peak
- **Type:** load
- **Requirement:** NFR-1, FR-4
- **Preconditions:** Staging sized like prod; catalog 50k SKUs × 60 stores; warm cache
- **Steps:** 1. k6/Gatling: 500 RPS browse mix (category/PDP/serviceability), 30 min
- **Expected:** Server p95 < 200 ms, p99 < 400 ms, error rate < 0.1%; Redis hit rate ≥ 95%
- **Test data:** `load profile `perf/browse.js``
- **Priority:** P0
