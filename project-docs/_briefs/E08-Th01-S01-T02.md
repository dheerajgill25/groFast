## Test cases this task must make pass

### TC-PERF-001 — Browse p95 < 200 ms at launch peak
- **Type:** load
- **Requirement:** NFR-1, FR-4
- **Preconditions:** Staging sized like prod; catalog 50k SKUs × 60 stores; warm cache
- **Steps:** 1. k6/Gatling: 500 RPS browse mix (category/PDP/serviceability), 30 min
- **Expected:** Server p95 < 200 ms, p99 < 400 ms, error rate < 0.1%; Redis hit rate ≥ 95%
- **Test data:** `load profile `perf/browse.js``
- **Priority:** P0

### TC-PERF-002 — Search p95 < 300 ms including typo queries
- **Type:** load
- **Requirement:** NFR-1, FR-5
- **Preconditions:** Meili seeded as prod-like
- **Steps:** 1. 100 RPS search, 20% typo-variant queries, 30 min
- **Expected:** p95 < 300 ms; relevance spot-checks still pass under load
- **Test data:** `query corpus `perf/search_terms.txt``
- **Priority:** P0

### TC-PERF-003 — Order confirm p95 < 3 s at surge
- **Type:** load
- **Requirement:** NFR-2, FR-12
- **Preconditions:** Gateway sandbox; full checkout path
- **Steps:** 1. Sustain 7 orders/min (10k/day avg) 2. Surge ×5 (35/min) for 15 min
- **Expected:** Pay-confirm→confirmed p95 < 3 s throughout; zero lost/duplicate orders; outbox lag < 5 s
- **Test data:** ``perf/checkout.js``
- **Priority:** P0

### TC-PERF-004 — 10× scale-out with zero code change
- **Type:** load
- **Requirement:** NFR-4
- **Preconditions:** Terraform/HPA config only
- **Steps:** 1. Scale replicas/partitions per runbook 2. Re-run TC-PERF-001..003 at 10× volumes (100k orders/day equivalent)
- **Expected:** All SLOs hold; scaling achieved via replica/partition counts only — no code or schema change required
- **Priority:** P1

### TC-PERF-005 — Tracking freshness ≤ 5 s at 10k concurrent sockets
- **Type:** load
- **Requirement:** NFR-7, FR-13
- **Preconditions:** 2k simulated riders @ 4 s GPS cadence; 10k customer/ops sockets
- **Steps:** 1. Run 30 min soak 2. Measure GPS-fix→client-render latency distribution
- **Expected:** p99 ≤ 5 s; no fan-out through Postgres/core (verified via metrics); socket reconnect storm (10% drop) recovers ≤ 30 s
- **Test data:** ``perf/tracking_sim.ts``
- **Priority:** P0

### TC-PERF-006 — Cold start < 3 s and offline grace on mid-range Android
- **Type:** manual-ops
- **Requirement:** NFR-9
- **Preconditions:** Physical mid-range device (e.g., 4 GB RAM class)
- **Steps:** 1. Cold-start ×10, measure 2. Load cart, enable airplane mode, kill app, reopen offline
- **Expected:** Median cold start < 3 s; cart + last catalog render from cache offline; syncs cleanly on reconnect
- **Test data:** `device matrix `perf/devices.md``
- **Priority:** P1
