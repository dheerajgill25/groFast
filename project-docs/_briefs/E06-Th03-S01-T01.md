## Test cases this task must make pass

### TC-FUNC-022 — Nearest online rider gets 30 s offer; accept assigns
- **Type:** integration
- **Requirement:** FR-18
- **Preconditions:** 3 online riders at known distances; order PACKED
- **Steps:** 1. Emit `order.packed` 2. Observe offers
- **Expected:** Nearest eligible rider offered first (Redis GEOSEARCH); 30 s TTL on offer; accept → assignment ACCEPTED, others never offered
- **Test data:** `rider GEO fixture `dispatch/riders.json``
- **Priority:** P0

### TC-FUNC-023 — Pickup verification + delivery proof enforced
- **Type:** e2e
- **Requirement:** FR-19, BR-8
- **Preconditions:** Accepted assignment
- **Steps:** 1. Rider arrives at store: OTP/scan pickup 2. Navigate 3. At drop capture OTP (or photo) 4. Complete
- **Expected:** Order DISPATCHED only after pickup verify; DELIVERED only after proof stored (object storage ref on assignment)
- **Priority:** P0

### TC-FUNC-024 — Online/offline toggle and accurate earnings
- **Type:** integration
- **Requirement:** FR-20
- **Preconditions:** Rider with 3 completed deliveries today
- **Steps:** 1. Toggle offline→online 2. Open earnings
- **Expected:** Toggle reflected in dispatch eligibility ≤ 5 s; per-order payout and daily total match assignment records exactly
- **Priority:** P1

### TC-FUNC-015 — [COND-COD §13] COD order within cap placed; rider sees collect amount
- **Type:** e2e
- **Requirement:** FR-11, BR-5
- **Preconditions:** COD enabled; order value ₹800 < cap ₹1,500
- **Steps:** 1. Choose COD 2. Place order 3. Open rider app on assignment
- **Expected:** Order placed, stock reserved at placement (no 15-min hold); rider app shows exact collect amount
- **Priority:** P1

### TC-NEG-023 — Delivery cannot complete without proof
- **Type:** integration
- **Requirement:** FR-19, BR-8
- **Preconditions:** Rider at drop
- **Steps:** 1. Attempt complete without OTP/photo 2. Attempt via raw API
- **Expected:** Both blocked; assignment stays ACCEPTED/arrived; with proof, completes normally
- **Priority:** P0

### TC-E2E-001 — Dark-store golden path: login → delivered → invoice → rate
- **Type:** e2e
- **Requirement:** FR-1..10,12,13,17,18,19,28; BR-2,8,10
- **Preconditions:** Staging full stack; sandbox gateway; test rider + picker
- **Steps:** 1. OTP login 2. Browse + search 3. Cart 4. Checkout with coupon, note ETA 5. Pay UPI 6. Picker packs 7. Rider accepts, picks up (OTP), delivers (proof) 8. Rate 5 stars
- **Expected:** Every transition correct and notified ≤ 10 s; tracking live ≤ 5 s; delivered within promised-ETA machinery; GST invoice in history; all events in audit + analytics
- **Test data:** `golden fixture `e2e/golden_path.md``
- **Priority:** P0

### TC-E2E-003 — [COND-COD §13] COD journey with collection + reconciliation
- **Type:** e2e
- **Requirement:** FR-11, FR-19; BR-5, BR-8
- **Preconditions:** COD enabled; cart under cap
- **Steps:** 1. Place COD order 2. Deliver with proof 3. Rider confirms collection 4. Check COD reconciliation report + rider cod_float
- **Expected:** Order requires proof AND collection confirmation to complete; collected amount hits reconciliation report and rider float; audit rows present
- **Priority:** P1
