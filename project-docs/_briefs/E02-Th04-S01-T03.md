## Parent story — E02-Th04-S01

**Title:** As a customer, I want a persistent single-store cart that refuses impossible quantities, so that my cart is always orderable

**User story:** As a customer, I want a persistent single-store cart that refuses impossible quantities, so that my cart is always orderable

**Acceptance criteria:**

- [ ] Cart is identical across app and web and survives session restart (Redis hot + PG durable)
- [ ] Quantity above stock or `max_per_order` blocks the increment with an explicit reason
- [ ] Mixed-store cart writes are rejected by the API, not just the UI (BR-1)

## Test cases this task must make pass

### TC-FUNC-009 — Cart add/step/remove with cross-device persistence
- **Type:** integration
- **Requirement:** FR-7
- **Preconditions:** Same user logged in on app + web
- **Steps:** 1. Add 2 SKUs on app 2. Step qty 3. Open web
- **Expected:** Running total correct; identical cart on second device; survives session restart (Redis hot + PG durable)
- **Priority:** P0

### TC-NEG-007 — Quantity above stock or per-order cap blocked with reason
- **Type:** unit
- **Requirement:** FR-7
- **Preconditions:** SKU: available 3, max_per_order 2
- **Steps:** 1. Step qty to 3 2. Then simulate stock 1, step to 2
- **Expected:** Increment blocked at cap with explicit reason ("max 2 per order" / "only 1 left"); cart never exceeds either limit
- **Priority:** P0

### TC-NEG-035 — Cross-store cart blocked
- **Type:** integration
- **Requirement:** BR-1
- **Preconditions:** Cart with items from store A
- **Steps:** 1. Switch location so serving store = B 2. Attempt add item
- **Expected:** Single-store rule enforced: prompt to clear/switch cart; API rejects mixed-store cart writes
- **Priority:** P0

### TC-E2E-002 — Partner-store order with stale-feed degradation
- **Type:** e2e
- **Requirement:** FR-23; BR-1, BR-6
- **Preconditions:** Partner store serving the zone; feed aged > 15 min then refreshed
- **Steps:** 1. Browse (observe "may take longer" while stale) 2. Refresh feed 3. Order 4. Partner confirms + marks READY 5. Dispatch → delivery
- **Expected:** Degraded listing while stale, fast ETA only when fresh; single-store cart (BR-1); partner notification + READY flow works; commission line recorded on order
- **Priority:** P0
