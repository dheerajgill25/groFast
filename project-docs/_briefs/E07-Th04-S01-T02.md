### TC-NEG-025 — Malformed bulk CSV rejected row-wise
- **Type:** integration
- **Requirement:** FR-21
- **Preconditions:** CSV: 5 valid rows, 3 malformed (bad price, missing SKU, bad category)
- **Steps:** 1. Upload
- **Expected:** Valid rows applied; 3 rejects reported with row number + reason; no partial-row corruption
- **Test data:** `fixture `catalog/bulk_bad.csv``
- **Priority:** P1

### TC-FUNC-026 — GRN, adjustment, cycle count produce audit trail + low-stock alert
- **Type:** integration
- **Requirement:** FR-22
- **Preconditions:** Dark store; SKU threshold = 10
- **Steps:** 1. GRN +50 2. Adjust −45 with reason 3. Observe
- **Expected:** Each mutation creates immutable audit record (actor/when/why); stock hits 5 → low-stock alert fires at threshold
- **Priority:** P0

### TC-NEG-026 — Stock mutation without reason/actor rejected
- **Type:** unit
- **Requirement:** FR-22
- **Preconditions:** Inventory API
- **Steps:** 1. POST adjustment omitting reason 2. Attempt adjustment driving qty below 0
- **Expected:** Both rejected 4xx; no audit row, no stock change; negative stock impossible
- **Priority:** P0

### TC-FUNC-027 — Partner feed ingests via API and CSV with rejection report
- **Type:** integration
- **Requirement:** FR-23
- **Preconditions:** Onboarded partner store; feed with 90 valid + 10 bad rows
- **Steps:** 1. Push API feed 2. Upload CSV feed 3. Read report
- **Expected:** Valid SKUs map/merge to master catalog; 10 rejects itemized with reasons; `last_ingested_at` updated, freshness = fresh
- **Test data:** `fixture `partners/feed_mixed.csv``
- **Priority:** P0

### TC-NEG-027 — Feed staler than 15 min degrades listing
- **Type:** integration
- **Requirement:** FR-23, BR-6, NFR-7
- **Preconditions:** Partner feed `last_ingested_at` 16 min ago
- **Steps:** 1. Browse as customer of that partner store 2. Check checkout ETA
- **Expected:** Store shows "delivery may take longer"; excluded from fast-ETA promises; wider ETA on order; recovers on fresh feed
- **Priority:** P0

### TC-FUNC-028 — Zone polygon publish updates serviceability ≤ 60 s
- **Type:** integration
- **Requirement:** FR-24

- **Steps:** 1. Extend polygon over P 2. Publish 3. New session at P
