## Test cases this task must make pass

### TC-FUNC-032 — Lifecycle transition dispatches mapped notification ≤ 10 s
- **Type:** integration
- **Requirement:** FR-28
- **Preconditions:** Templates registered (DLT sandbox)
- **Steps:** 1. Drive order through PLACED→PACKED→DISPATCHED→DELIVERED
- **Expected:** Each transition's mapped push sent ≤ 10 s of commit; notification rows recorded per event
- **Priority:** P0

### TC-NEG-032 — Push fails → SMS fallback; both fail → retry + alert
- **Type:** integration
- **Requirement:** FR-28
- **Preconditions:** FCM sandbox forced-fail for target device
- **Steps:** 1. Drive one transition 2. Then force SMS fail too
- **Expected:** SMS fallback row created (`fallback_of` set) ≤ 10 s; on double failure, retry with backoff + ops alert; no silent loss
- **Priority:** P0

### TC-INT-003 — Consumers are idempotent under duplicate delivery
- **Type:** integration
- **Requirement:** FR-28, Arch §3a C-9
- **Preconditions:** `order.placed` event in topic
- **Steps:** 1. Redeliver the same event 3× to notification, audit, picker-projection consumers
- **Expected:** Exactly one notification, one audit row, one picker-queue entry; duplicates detected via processed-event key and skipped
- **Priority:** P0

### TC-E2E-001 — Dark-store golden path: login → delivered → invoice → rate
- **Type:** e2e
- **Requirement:** FR-1..10,12,13,17,18,19,28; BR-2,8,10
- **Preconditions:** Staging full stack; sandbox gateway; test rider + picker
- **Steps:** 1. OTP login 2. Browse + search 3. Cart 4. Checkout with coupon, note ETA 5. Pay UPI 6. Picker packs 7. Rider accepts, picks up (OTP), delivers (proof) 8. Rate 5 stars
- **Expected:** Every transition correct and notified ≤ 10 s; tracking live ≤ 5 s; delivered within promised-ETA machinery; GST invoice in history; all events in audit + analytics
