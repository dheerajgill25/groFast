## Test cases this task must make pass

### TC-FUNC-017 — Live tracking updates ≤ 5 s with instant status pushes
- **Type:** e2e
- **Requirement:** FR-13, NFR-7
- **Preconditions:** Dispatched order; rider app streaming GPS every 4 s
- **Steps:** 1. Open tracking 2. Move rider (mock GPS) 3. Transition status
- **Expected:** Map position refreshes ≤ 5 s from GPS fix; PACKED→DISPATCHED→DELIVERED push instantly on the same socket
- **Priority:** P0

### TC-NEG-017 — WebSocket loss degrades to HTTPS polling
- **Type:** integration
- **Requirement:** FR-13, NFR-3
- **Preconditions:** Tracking session live
- **Steps:** 1. Sever WS (network blip) 2. Keep screen open
- **Expected:** Client falls back to `GET /v1/orders/{id}/track`; last-known position from Redis shown; no error screen
- **Priority:** P0

### TC-INT-010 — WebSocket reconnect resumes tracking with missed events
- **Type:** integration
- **Requirement:** FR-13, Arch §5d
- **Preconditions:** Active tracking session
- **Steps:** 1. Kill socket 2. Transition order status during the gap 3. Client reconnects
- **Expected:** Client resubscribes to `track:{order_id}`; current status + last position delivered on reconnect (state fetch on subscribe); no permanent stale screen
- **Priority:** P0

### TC-INT-011 — Realtime gateway pod kill leaves core path unharmed
- **Type:** manual-ops
- **Requirement:** NFR-3, Arch §4
- **Preconditions:** ≥ 2 realtime pods; live tracking sessions; steady checkout load
- **Steps:** 1. Kill one realtime pod
- **Expected:** Sockets reconnect to surviving pod ≤ 10 s; checkout p95 unaffected during the event; no order failures
- **Test data:** `chaos runbook`
- **Priority:** P1

### TC-PERF-005 — Tracking freshness ≤ 5 s at 10k concurrent sockets
- **Type:** load
- **Requirement:** NFR-7, FR-13
- **Preconditions:** 2k simulated riders @ 4 s GPS cadence; 10k customer/ops sockets
- **Steps:** 1. Run 30 min soak 2. Measure GPS-fix→client-render latency distribution
- **Expected:** p99 ≤ 5 s; no fan-out through Postgres/core (verified via metrics); socket reconnect storm (10% drop) recovers ≤ 30 s
- **Test data:** ``perf/tracking_sim.ts``
- **Priority:** P0
