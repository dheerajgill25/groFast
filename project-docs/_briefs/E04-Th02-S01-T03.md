## Parent story — E04-Th02-S01

**Title:** As a rider, I want nearby jobs offered to me automatically, so that I earn without hunting

**User story:** As a rider, I want nearby jobs offered to me automatically, so that I earn without hunting

**Acceptance criteria:**

- [ ] Nearest eligible online rider is offered first via Redis `GEOSEARCH`, 30 s TTL on the offer
- [ ] Declines/timeouts cascade to next-nearest; exhaustion lands in the ops exception queue for force-assign
- [ ] A rider with an active order (BR-7) or offline is never offered
- [ ] Exactly one expiry handling even if the dispatch worker restarts mid-offer

**Linked test cases:** TC-FUNC-022, TC-NEG-022, TC-NEG-024, TC-NEG-037, TC-INT-016

## Test cases this task must make pass

### TC-FUNC-022 — Nearest online rider gets 30 s offer; accept assigns
- **Type:** integration
- **Requirement:** FR-18
- **Preconditions:** 3 online riders at known distances; order PACKED
- **Steps:** 1. Emit `order.packed` 2. Observe offers
- **Expected:** Nearest eligible rider offered first (Redis GEOSEARCH); 30 s TTL on offer; accept → assignment ACCEPTED, others never offered
- **Test data:** `rider GEO fixture `dispatch/riders.json``
- **Priority:** P0

### TC-NEG-022 — Offer declines/timeouts cascade; exhaustion hits exception queue
- **Type:** integration
- **Requirement:** FR-18
- **Preconditions:** 2 online riders; both decline
- **Steps:** 1. Emit `order.packed` 2. Decline offer 1 3. Let offer 2 time out (30 s)
- **Expected:** Cascade nearest→next; on exhaustion order lands in exception queue for force-assign; ops force-assign works
- **Priority:** P0

### TC-NEG-024 — Offline rider receives no offers
- **Type:** integration
- **Requirement:** FR-20
- **Preconditions:** Rider toggled offline; only rider in zone
- **Steps:** 1. Emit `order.packed`
- **Expected:** Rider excluded from GEOSEARCH candidates; order goes to exception queue (no eligible rider), not to offline rider
- **Priority:** P1

### TC-NEG-037 — Rider with active order excluded from dispatch
- **Type:** integration
- **Requirement:** BR-7
- **Preconditions:** Rider R1 has ACCEPTED assignment; R1 is nearest to new order
- **Steps:** 1. Emit `order.packed` for second order
- **Expected:** R1 never offered (max 1 active order); next-nearest rider offered instead
- **Priority:** P0

### TC-INT-016 — Offer TTL expiry cascades exactly once under worker restart
- **Type:** integration
- **Requirement:** FR-18
- **Preconditions:** Offer OFFERED with 30 s TTL; dispatch worker restarted at T+15 s
- **Steps:** 1. Restart worker mid-offer 2. Let TTL lapse
- **Expected:** Exactly one expiry handling: single next-rider offer, never two concurrent active offers for one order (idempotent timer/state guard)
- **Priority:** P0
