## Parent story — E05-Th01-S01

**Title:** As an ops manager, I want GRN, adjustments and cycle counts with a full audit trail and alerts, so that system stock matches shelf stock

**User story:** As an ops manager, I want GRN, adjustments and cycle counts with a full audit trail and alerts, so that system stock matches shelf stock

**Acceptance criteria:**

- [ ] Every stock mutation produces exactly one immutable audit row (who/when/why); reason is mandatory
- [ ] Negative stock is impossible; adjustments without reason/actor are rejected 4xx with no stock change
- [ ] Low-stock alerts fire at the configurable threshold
- [ ] Stock changes reach the storefront cache and search index within 60 s

**Linked FRs:** FR-22, FR-21
**Linked test cases:** TC-FUNC-026, TC-NEG-026, TC-SEC-009, TC-INT-015, TC-NEG-004
**Linked screens:** SC-035, SC-036

## Test cases this task must make pass

### TC-FUNC-026 — GRN, adjustment, cycle count produce audit trail + low-stock alert
- **Type:** integration
- **Requirement:** FR-22
- **Preconditions:** Dark store; SKU threshold = 10
- **Steps:** 1. GRN +50 2. Adjust −45 with reason 3. Observe
- **Expected:** Each mutation creates immutable audit record (actor/when/why); stock hits 5 → low-stock alert fires at threshold
- **Priority:** P0

### TC-NEG-026 — Stock mutation without reason/actor rejected
- **Type:** unit
- **Requirement:** FR-22
- **Preconditions:** Inventory API
- **Steps:** 1. POST adjustment omitting reason 2. Attempt adjustment driving qty below 0
- **Expected:** Both rejected 4xx; no audit row, no stock change; negative stock impossible
- **Priority:** P0

### TC-SEC-009 — Audit completeness matrix: every money/stock mutation audited
- **Type:** security
- **Requirement:** NFR-10, FR-22
- **Preconditions:** Staging with all flows executable
- **Steps:** 1. Execute one of each: GRN, adjustment, cycle count, reserve, release, price change, coupon apply, payment capture, refund, COD collection [COND-COD §13], commission record 2. Query `audit_event`
- **Expected:** 100%: each mutation has exactly one append-only audit row with actor, actor_role, before/after, reason; UPDATE/DELETE on `audit_event` blocked at DB level
- **Test data:** `mutation checklist `sec/audit_matrix.md``
- **Priority:** P0

### TC-INT-015 — Stock event propagates to cache + search index ≤ 60 s
- **Type:** integration
- **Requirement:** NFR-7, FR-21
- **Preconditions:** SKU in Redis cache and Meili index
- **Steps:** 1. Zero its stock via inventory API 2. Poll storefront + search
- **Expected:** Cache invalidated and Meili updated ≤ 60 s; SKU stops rendering in browse and search within the window
- **Priority:** P0

### TC-NEG-004 — OOS/unpriced SKUs never render in category
- **Type:** integration
- **Requirement:** FR-4
- **Preconditions:** SKU flips to OOS mid-session
- **Steps:** 1. Browse category 2. Invalidate stock 3. Refresh after propagation
- **Expected:** OOS SKU absent (or marked, per design) after ≤ 60 s propagation; no ghost inventory purchasable
- **Priority:** P0
