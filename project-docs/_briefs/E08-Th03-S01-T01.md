## Test cases this task must make pass

### TC-PERF-007 — Chaos: pod kill + AZ loss keeps order path alive
- **Type:** manual-ops
- **Requirement:** NFR-3
- **Preconditions:** Staging multi-AZ; steady checkout load running
- **Steps:** 1. Kill a core pod 2. Then simulate full-AZ outage (cordon+drain AZ)
- **Expected:** No failed orders (retries absorb); order-path availability ≥ 99.9% over the window; managed-Postgres failover transparent
- **Test data:** `chaos runbook `ops/chaos.md``
- **Priority:** P0

### TC-PERF-008 — Graceful degradation via kill switches
- **Type:** manual-ops
- **Requirement:** NFR-3
- **Preconditions:** Feature flags wired for ratings, analytics ingestion, partner ingestion
- **Steps:** 1. Flip each flag off under load 2. Sever Redpanda from one consumer group
- **Expected:** Order path (browse→checkout→track) fully functional throughout; tracking falls back to polling if realtime shed; consumers resume from offset with zero event loss
- **Priority:** P1

### TC-INT-011 — Realtime gateway pod kill leaves core path unharmed
- **Type:** manual-ops
- **Requirement:** NFR-3, Arch §4
- **Preconditions:** ≥ 2 realtime pods; live tracking sessions; steady checkout load
- **Steps:** 1. Kill one realtime pod
- **Expected:** Sockets reconnect to surviving pod ≤ 10 s; checkout p95 unaffected during the event; no order failures
- **Test data:** `chaos runbook`
- **Priority:** P1

### TC-INT-002 — Outbox relay is at-least-once across crashes
- **Type:** integration
- **Requirement:** Arch §2a
- **Preconditions:** Order committed; relay killed after commit, before publish
- **Steps:** 1. Kill relay process at the crash point 2. Restart
- **Expected:** Event published to Redpanda after restart; `published_at` set; no event ever lost between commit and publish
- **Test data:** `fault-injection harness`
- **Priority:** P0

### TC-INT-004 — Full topic replay rebuilds projections without duplicate side effects
- **Type:** integration
- **Requirement:** Arch §2a, NFR-10
- **Preconditions:** Picker/analytics projections dropped
- **Steps:** 1. Reset consumer group to offset 0 2. Replay day of events
- **Expected:** Projections rebuilt byte-consistent with Postgres truth; side-effecting consumers (SMS/push/refund) emit nothing on replay
- **Test data:** `seeded topic fixture`
- **Priority:** P1

### TC-NEG-017 — WebSocket loss degrades to HTTPS polling
- **Type:** integration
- **Requirement:** FR-13, NFR-3
- **Preconditions:** Tracking session live
- **Steps:** 1. Sever WS (network blip) 2. Keep screen open
- **Expected:** Client falls back to `GET /v1/orders/{id}/track`; last-known position from Redis shown; no error screen
- **Priority:** P0
