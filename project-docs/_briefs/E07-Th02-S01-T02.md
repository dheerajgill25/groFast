## Test cases this task must make pass

### TC-FUNC-029 — ETA breach lands in exception queue ≤ 30 s with one-click actions
- **Type:** integration
- **Requirement:** FR-25
- **Preconditions:** Order with `eta_promised_at` in the past, not delivered
- **Steps:** 1. Let clock pass promise 2. Watch queue
- **Expected:** Breach row appears ≤ 30 s; reassign-rider / call-customer / cancel-refund actions present and functional
- **Priority:** P0

### TC-NEG-029 — Exception action on terminal-state order rejected
- **Type:** integration
- **Requirement:** FR-25
- **Preconditions:** Order already DELIVERED still visible in stale queue view
- **Steps:** 1. Click reassign-rider
- **Expected:** Action refused with state reason; queue row refreshes/clears; no zombie assignment created
- **Priority:** P2

### TC-FUNC-020 — Low rating triggers reason picker and ops exception
- **Type:** integration
- **Requirement:** FR-16
- **Preconditions:** Delivered order
- **Steps:** 1. Rate 2 stars 2. Pick reason 3. Check ops queue
- **Expected:** Reason picker appears at ≤ 3 stars; record lands in exception queue with order link
- **Priority:** P2

### TC-NEG-022 — Offer declines/timeouts cascade; exhaustion hits exception queue
- **Type:** integration
- **Requirement:** FR-18
- **Preconditions:** 2 online riders; both decline
- **Steps:** 1. Emit `order.packed` 2. Decline offer 1 3. Let offer 2 time out (30 s)
- **Expected:** Cascade nearest→next; on exhaustion order lands in exception queue for force-assign; ops force-assign works
- **Priority:** P0
