## Parent story — E03-Th01-S02

**Title:** As a customer, I want to cancel, reorder and rate, so that mistakes are cheap and repeat shopping is instant

**User story:** As a customer, I want to cancel, reorder and rate, so that mistakes are cheap and repeat shopping is instant

**Acceptance criteria:**

- [ ] Cancel before dispatch releases stock and auto-initiates the refund; cancel after dispatch is refused via app **and** raw API
- [ ] Reorder adds available items at current price and flags unavailable ones — never silently
- [ ] Ratings ≤ 3 stars open a reason picker and land in the ops exception queue; non-delivered orders cannot be rated

**Linked FRs:** FR-14, FR-15, FR-16

## Test cases this task must make pass

### TC-FUNC-018 — Cancel before dispatch releases stock and auto-refunds
- **Type:** integration
- **Requirement:** FR-14, BR-4
- **Preconditions:** Prepaid order in PACKED
- **Steps:** 1. Customer cancels 2. Inspect inventory, refund row
- **Expected:** Reserved qty released; `refund` row created immediately with `sla_deadline` = +24 h; gateway refund API called with idempotency key = refund ID; no manual approval (amount < ₹2,000)
- **Test data:** `order value ₹499`
- **Priority:** P0

### TC-FUNC-019 — One-tap reorder with unavailable items flagged
- **Type:** integration
- **Requirement:** FR-15
- **Preconditions:** Past order: SKU-1 still stocked, SKU-2 now OOS
- **Steps:** 1. Tap reorder
- **Expected:** SKU-1 lands in cart at current price; SKU-2 flagged unavailable, not added silently
- **Priority:** P1

### TC-FUNC-020 — Low rating triggers reason picker and ops exception
- **Type:** integration
- **Requirement:** FR-16
- **Preconditions:** Delivered order
- **Steps:** 1. Rate 2 stars 2. Pick reason 3. Check ops queue
- **Expected:** Reason picker appears at ≤ 3 stars; record lands in exception queue with order link
- **Priority:** P2

### TC-NEG-018 — Cancel after dispatch is blocked
- **Type:** integration
- **Requirement:** FR-14
- **Preconditions:** Order DISPATCHED
- **Steps:** 1. Attempt cancel via app and via API
- **Expected:** Both refused with reason; order state unchanged; support path (FR-26) referenced
- **Priority:** P0

### TC-NEG-019 — Reorder with everything unavailable
- **Type:** integration
- **Requirement:** FR-15
- **Preconditions:** Past order; all SKUs now delisted
- **Steps:** 1. Tap reorder
- **Expected:** Zero items added; clear "all unavailable" message; no empty-cart checkout possible
- **Priority:** P2

### TC-NEG-020 — Rating rejected for non-delivered order
- **Type:** unit
- **Requirement:** FR-16
- **Preconditions:** Order in DISPATCHED
- **Steps:** 1. POST rating via API
- **Expected:** 4xx with reason; no rating row; no exception-queue entry
- **Priority:** P2

### TC-E2E-004 — Cancellation + refund lands within SLA on gateway sandbox
- **Type:** e2e
- **Requirement:** FR-14; BR-4
- **Preconditions:** Prepaid order in PACKED
- **Steps:** 1. Cancel 2. Track refund through gateway sandbox webhooks to terminal state
- **Expected:** Stock released instantly; refund initiated immediately (< 24 h SLA with 20 h alert never firing); refund webhook closes loop; customer notified at each step
- **Priority:** P0
