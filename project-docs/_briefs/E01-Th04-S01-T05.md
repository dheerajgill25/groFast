## Test cases this task must make pass

### TC-INT-001 — Outbox rows commit atomically with the state change
- **Type:** integration
- **Requirement:** FR-12, NFR-10, Arch §5a
- **Preconditions:** Checkout ready
- **Steps:** 1. Confirm payment (success run) 2. Force txn rollback on a second run (inject line shortfall)
- **Expected:** Success: order + `order.placed`/`payment.captured`/`stock.reserved` outbox rows in one commit. Rollback: zero outbox rows — an event can never exist without its state change
- **Priority:** P0

### TC-INT-002 — Outbox relay is at-least-once across crashes
- **Type:** integration
- **Requirement:** Arch §2a
- **Preconditions:** Order committed; relay killed after commit, before publish
- **Steps:** 1. Kill relay process at the crash point 2. Restart
- **Expected:** Event published to Redpanda after restart; `published_at` set; no event ever lost between commit and publish
- **Test data:** `fault-injection harness`
- **Priority:** P0

### TC-INT-003 — Consumers are idempotent under duplicate delivery
- **Type:** integration
- **Requirement:** FR-28, Arch §3a C-9
- **Preconditions:** `order.placed` event in topic
- **Steps:** 1. Redeliver the same event 3× to notification, audit, picker-projection consumers
- **Expected:** Exactly one notification, one audit row, one picker-queue entry; duplicates detected via processed-event key and skipped
- **Priority:** P0

### TC-INT-004 — Full topic replay rebuilds projections without duplicate side effects
- **Type:** integration
- **Requirement:** Arch §2a, NFR-10
- **Preconditions:** Picker/analytics projections dropped
- **Steps:** 1. Reset consumer group to offset 0 2. Replay day of events
- **Expected:** Projections rebuilt byte-consistent with Postgres truth; side-effecting consumers (SMS/push/refund) emit nothing on replay
- **Test data:** `seeded topic fixture`
- **Priority:** P1

### TC-INT-020 — Money-path writes without Idempotency-Key rejected
- **Type:** integration
- **Requirement:** FR-10, Arch C-6
- **Preconditions:** API gateway enforcing idempotency on writes
- **Steps:** 1. POST checkout-confirm/refund endpoints without the header
- **Expected:** 400 with contract error; with header, retries dedupe; non-money reads unaffected
- **Priority:** P1
